# Contribuer

## Règles

- Utiliser des exemples anonymisés.
- Ne jamais ajouter de secrets ou de dumps.
- Citer les sources officielles.
- Indiquer la date de vérification des interfaces.
- Tester les commandes dans un environnement non critique.
- Signaler clairement les opérations irréversibles.

## Proposer une correction

1. Créer une branche.
2. Modifier la documentation.
3. Vérifier les liens.
4. Ouvrir une Pull Request.
5. Décrire l’interface et la date observées.
